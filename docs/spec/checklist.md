状态说明：`未开始` / `进行中` / `已完成` / `需澄清`。

## 功能需求逐项检查（需求 → 宪法）

### 需求 1 - 账号注册与登录
- [ ] 第三方登录（Apple/Google/微信）认证失败可重试流程已定义且文案英文化。  
- [ ] 登录态建立、退出、失效策略满足“默认安全 + 最小权限”。  
- [ ] 身份相关日志不包含邮箱验证码、令牌等敏感信息。  
- [ ] 对应测试覆盖邮箱登录、第三方登录、失败分支与中断分支。

### 需求 2 - 订阅源发现与添加
- [ ] “公开目录推荐 + RSS URL 添加”双入口都可用。  
- [ ] URL 校验失败、网络失败存在明确错误提示与重试入口。  
- [ ] 订阅/取消订阅行为结果一致，跨页面状态同步一致。  
- [ ] 订阅数据遵循“用户数据隔离”，不可越权访问他人订阅。

### 需求 3 - 今日聚合流
- [ ] 时间范围与排序切换行为稳定、可回放、可测试。  
- [ ] 从今日页进入阅读后返回，保持原筛选与滚动状态。  
- [ ] 文章列表渲染在目标机型保持流畅。  
- [ ] 收藏动作与书架状态保持一致。

### 需求 4 - 书架与收藏管理
- [ ] 未读数与更新时间计算规则一致且可追溯。  
- [ ] 分类筛选语义在“今日/书架”保持统一。  
- [ ] 空状态（探索目录）可引导恢复操作。  
- [ ] 源详情与文章详情导航链路一致。

### 需求 5 - 文章阅读体验
- [ ] 字体、主题、翻译语言偏好可持久化并跨会话生效。  
- [ ] 朗读、翻译均按 P0 能力交付，失败时有降级提示。  
- [ ] 阅读进度更新机制稳定，切后台后恢复正确。  
- [ ] 收藏/分享行为与系统能力边界一致、错误可恢复。

### 需求 6 - 个人中心与偏好设置
- [ ] 账户统计口径统一（订阅、已读、收藏定义一致）。  
- [ ] 偏好设置更新后在阅读页实时生效。  
- [ ] 退出登录清理会话与本地敏感缓存。  
- [ ] 通知与语言设置变更有可验证结果。

### 需求 7 - 导航与信息架构
- [ ] 底部导航 4 个一级入口状态高亮一致。  
- [ ] 详情返回行为满足“来源页面状态保持”要求。  
- [ ] 首次进入默认页为“今日”。  
- [ ] 同类导航动作在不同入口行为一致。

### 需求 8 - 非功能与视觉一致性
- [ ] 统一设计令牌在核心页面一致执行。  
- [ ] 页面分组优先色块层次，不使用割裂式分割线。  
- [ ] 网络失败与超时均具备可恢复机制。  
- [ ] 离线缓存能力可用：已缓存文章可在离线查看。  
- [ ] 离线内容边界清晰（可读内容、不可读内容、刷新策略）。

## 宪法执行清单（开发前/开发中/发布前）

### 开发前
- [ ] 需求无歧义，关键流程已有澄清记录。  
- [ ] 安全与隐私边界已评审（鉴权、数据隔离、日志脱敏）。  
- [ ] 测试点已拆分并形成测试计划。  
- [ ] 新增依赖必要性评估完成。

### 开发中
- [ ] 按 TDD 循环推进（Red → Green → Refactor）。  
- [ ] 模块化与统一接口持续检查，避免重复实现。  
- [ ] 关键变更记录“背景-选择-影响”。  
- [ ] UI 变更进行视觉回归核验。

### 发布前
- [ ] Lint、测试、类型检查全部通过。  
- [ ] 关键路径回归通过（登录、订阅、阅读、离线）。  
- [ ] 安全扫描与质量门禁通过。  
- [ ] 文档同步更新（需求、实现说明、测试记录）。
