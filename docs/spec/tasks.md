# 实施计划

执行时需严格遵循 `docs/spec/requirements.md` 中对应需求的验收标准。每项任务通过 `_需求:` 引用相关需求编号。

- [x] 1. 基础环境搭建：提交前强制检查（Husky + lint-staged）
- 具体要做的事情：安装并配置 Husky 与 lint-staged，接入 `pre-commit` 钩子，提交前自动执行代码格式化、Lint、类型检查与关键测试，校验失败时阻止提交。
- 验证方式：单元测试（脚本）+ 本地提交冒烟（故意制造 lint 错误验证拦截）。
- \_需求: requirements.md 需求9(3,4)、design.md 7.1, 7.2。

- [x] 2. 基础环境搭建：CI/CD 集成 GitHub Actions（含代码审计与安全扫描）
- 具体要做的事情：建立 GitHub Actions 工作流，覆盖安装依赖、构建、测试、Lint、类型检查；并集成依赖漏洞扫描、SAST/代码审计与密钥泄漏扫描，配置 PR 与主分支触发及失败阻断策略。
- 验证方式：CI 集成测试（PR 流水线）+ 安全扫描报告校验。
- \_需求: requirements.md 需求9(3,4,8)、design.md 8, 9。

- [ ] 3. 搭建认证路由守卫与未登录拦截
- 具体要做的事情：实现全局会话状态、首启登录页跳转、业务页访问拦截、登录后默认进入今日页。
- 验证方式：E2E（首启、未登录访问、登录后回跳）。
- \_需求: requirements.md 需求1(10,11)、需求8(4)。

- [ ] 4. 实现邮箱验证码发送能力
- 具体要做的事情：实现发送验证码接口调用、倒计时组件、重发节流与失败提示。
- 验证方式：单元测试 + 集成测试（倒计时/重发状态机）。
- \_需求: requirements.md 需求1(1)。

- [ ] 5. 实现验证码校验与注册凭证流转
- 具体要做的事情：实现验证码校验接口、校验成功后进入密码设置页、失败提示与过期处理。
- 验证方式：E2E（正确码、错误码、过期码）。
- \_需求: requirements.md 需求1(2,3)。

- [ ] 6. 实现邮箱密码注册与登录
- 具体要做的事情：实现条款勾选校验、密码注册提交、邮箱密码登录、错误凭证提示。
- 验证方式：集成测试 + E2E（注册成功、登录成功、错误凭证）。
- \_需求: requirements.md 需求1(4,5,6,7)。

- [ ] 7. 接入第三方登录与异常处理
- 具体要做的事情：接入 Apple/Google/微信登录，处理用户取消授权、失败重试、邮箱补充与合并异常提示。
- 验证方式：集成测试（provider mock）+ E2E 冒烟。
- \_需求: requirements.md 需求1(8,9,12,13,14)。

- [ ] 8. 实现发现页目录查询与筛选
- 具体要做的事情：实现公开目录列表、分类筛选、关键词搜索、分页加载与状态回显。
- 验证方式：集成测试 + 视觉回归（发现页）。
- \_需求: requirements.md 需求2(1,2)。

- [ ] 9. 实现 RSS URL 导入与订阅状态切换
- 具体要做的事情：实现 URL 校验导入、订阅/取消订阅幂等处理、失败原因提示。
- 验证方式：单元测试 + 集成测试（合法/非法/不可达 URL）。
- \_需求: requirements.md 需求2(3,4,5,6)。

- [ ] 10. 实现 free 用户订阅上限门禁
- 具体要做的事情：在应用层和服务端双重校验订阅上限 10，命中后阻断并提示升级会员。
- 验证方式：集成测试（10->11 边界）+ E2E。
- \_需求: requirements.md 需求2(7)、需求7(3,4)。

- [ ] 11. 实现今日聚合流与时间范围切换
- 具体要做的事情：实现今日页列表、时间范围过滤（今日/昨天/本周）、发布时间倒序渲染。
- 验证方式：单元测试（排序/过滤）+ E2E。
- \_需求: requirements.md 需求3(1,2,3)。

- [ ] 12. 实现精选推荐流（精选栏目过滤 + 时间倒序）
- 具体要做的事情：按精选订阅栏目筛选文章并按发布时间倒序返回，确保与今日流排序规则一致。
- 验证方式：集成测试（过滤与排序一致性）。
- \_需求: requirements.md 需求3(6)。

- [ ] 13. 打通文章跳转与收藏联动
- 具体要做的事情：实现今日/书架到阅读页跳转，收藏状态即时切换并在书架可见。
- 验证方式：E2E（跨页状态一致性）。
- \_需求: requirements.md 需求3(4,5)、需求4(5)、需求5(2)。

- [ ] 14. 实现书架列表、分类与未读计数展示
- 具体要做的事情：实现订阅源列表、分类过滤、未读数与最近更新时间展示、底部探索目录入口。
- 验证方式：集成测试 + 视觉回归（书架页）。
- \_需求: requirements.md 需求4(1,2,3,4)。

- [ ] 15. 实现阅读页基础渲染与偏好持久化
- 具体要做的事情：渲染标题/来源/时间/时长/正文，支持字体、主题、行高即时切换并持久化。
- 验证方式：单元测试（偏好存储）+ 视觉回归（阅读页）。
- \_需求: requirements.md 需求5(1,3,4)、需求6(2)。

- [ ] 16. 实现阅读进度与系统分享
- 具体要做的事情：实现进度上报与底部进度展示，实现系统分享调用与失败兜底提示。
- 验证方式：集成测试 + E2E。
- \_需求: requirements.md 需求5(2,7)。

- [ ] 17. 实现朗读/翻译会员门禁与执行链路
- 具体要做的事情：实现 premium 才可用的朗读与翻译接口调用，free 用户阻断与升级引导。
- 验证方式：单元测试（门禁）+ 集成测试（能力调用）。
- \_需求: requirements.md 需求5(5,6,8)、需求7(3,4)。

- [ ] 18. 实现个人中心与偏好设置页面
- 具体要做的事情：展示账户信息与统计，支持界面语言和默认翻译语言设置，支持退出登录。
- 验证方式：集成测试 + E2E。
- \_需求: requirements.md 需求6(1,4,5,6)。

- [ ] 19. 实现通知设置入口占位
- 具体要做的事情：在首版提供通知设置入口与说明文案，不接入通知开关后端能力。
- 验证方式：视觉回归 + E2E 冒烟。
- \_需求: requirements.md 需求6(3)。

- [ ] 20. 实现会员购买与状态同步
- 具体要做的事情：实现套餐展示、创建支付会话、支付回调幂等、会员状态刷新与失效降级。
- 验证方式：集成测试（webhook 幂等）+ E2E（升级/失效）。
- \_需求: requirements.md 需求7(1,2,3,4)。

- [ ] 21. 实现底部导航与返回态保持
- 具体要做的事情：实现一级导航切换与高亮、详情返回原列表并恢复滚动位置和筛选状态。
- 验证方式：E2E（多入口返回态）。
- \_需求: requirements.md 需求8(1,2,3,4)。

- [ ] 22. 实现离线缓存与离线读
- 具体要做的事情：进入文章详情触发正文与图片缓存，离线时可读取缓存正文/图片/元数据。
- 验证方式：集成测试（缓存命中）+ E2E（飞行模式场景）。
- \_需求: requirements.md 需求9(5,6)。

- [ ] 23. 实现离线写阻断与通用失败语义
- 具体要做的事情：客户端离线阻断收藏/订阅/偏好写操作；异常到达服务端时返回通用写失败语义。
- 验证方式：单元测试（客户端门禁）+ 集成测试（服务端错误语义）。

- [ ] 24. 落地视觉一致性与主题令牌
- 具体要做的事情：统一颜色/字体/圆角/层级令牌，页面分组优先使用色块层次，统一错误与重试组件。
- 验证方式：视觉回归（今日/发现/书架/阅读/我的）。

- [ ] 25. 完成性能指标优化
- 具体要做的事情：优化首屏数据加载、列表虚拟化与图片懒加载，确保今日首屏与核心列表流畅度达标。

- \_需求: requirements.md 需求9(3,9,10)。

- [ ] 26. 完成可访问性适配
- 具体要做的事情：支持 200% 动态字体、关键控件读屏名称、文本对比度满足 WCAG AA。

- \_需求: requirements.md 需求9(11,12,13)。

- [ ] 27. 补齐自动化测试矩阵与发布回归
- 具体要做的事情：为每个需求至少补 1 条自动化验收，覆盖登录->订阅->阅读->会员->离线关键路径。
- 验证方式：单元测试 + 集成测试 + E2E + 视觉回归全量通过。
