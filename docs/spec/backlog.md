# 研发任务 Backlog（OneRss）

## 1. 说明

本 Backlog 基于以下定稿文档：

- `docs/spec/requirements.md`
- `docs/spec/constitution.md`
- `docs/spec/design.md`
- `docs/spec/data.md`
- `docs/spec/api.md`

任务按里程碑 `M1 -> M5` 组织，并标注优先级（P0/P1/P2）、责任域（前端/后端/数据/测试）与验收映射（需求编号）。

## 2. 优先级定义

- **P0**：上线阻断项，不完成不可进入下阶段。
- **P1**：高优先增强项，可在同里程碑末尾完成。
- **P2**：优化项，可进入后续迭代。

## 3. 里程碑总览

| 里程碑 | 目标                 | 关联需求                 | 交付结果                     |
| ------ | -------------------- | ------------------------ | ---------------------------- |
| M1     | 认证闭环与导航壳     | 需求1、需求8             | 可登录进入主流程，未登录拦截 |
| M2     | 发现订阅与书架可用   | 需求2、需求4             | 可发现、可订阅、可管理书架   |
| M3     | 今日流与阅读基础闭环 | 需求3、需求5（基础）     | 可阅读、可收藏、可记录进度   |
| M4     | 会员计费与能力门禁   | 需求7、需求5（高级能力） | 月付/年付、翻译朗读门禁生效  |
| M5     | 离线与非功能验收     | 需求9                    | 离线可读、性能与可访问性达标 |

## 4. 详细任务清单

## M1 - 认证与导航壳（P0）

### M1-P0-01 认证路由守卫

- **优先级**：P0
- **责任域**：前端
- **描述**：实现未登录拦截，首次启动进入注册登录页；登录后进入主导航。
- **验收映射**：需求1.8、1.9、需求8.4
- **完成定义**：
  - 未登录访问业务页自动跳转登录
  - 登录成功落在“今日”页
  - 退出登录后会话清理并回登录入口

### M1-P0-02 邮箱验证码登录流程

- **优先级**：P0
- **责任域**：前端 + 后端
- **描述**：实现发送验证码、校验验证码、错误提示与重试流程。
- **验收映射**：需求1.1~1.5
- **完成定义**：
  - 验证码成功与失败分支可验证
  - 错误提示文案为英文
  - 失败后可重试

### M1-P0-03 第三方登录与账号合并

- **优先级**：P0
- **责任域**：后端 + 前端
- **描述**：接入 Apple/Google/WeChat 登录，按规则完成自动合并或绑定引导。
- **验收映射**：需求1.6、1.7、1.10、1.11、1.12
- **完成定义**：
  - 同邮箱自动合并
  - 无邮箱需补充并验证后完成
  - 冲突异常返回绑定引导

### M1-P1-01 登录审计日志基础

- **优先级**：P1
- **责任域**：后端
- **描述**：记录关键登录事件（成功/失败/合并冲突）。
- **验收映射**：宪法-安全与隐私（可审计性）

### M1 测试任务

- **T1**（P0，测试）：认证主流程 E2E（邮箱/第三方/退出登录）
- **T2**（P0，测试）：未登录拦截回归
- **T3**（P1，测试）：账号合并冲突场景回归
